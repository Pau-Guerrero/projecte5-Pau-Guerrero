# 📘 README — Nou flux de treball amb Git + GitHub

Fins ara hem treballat directament des de la web de GitHub. Tot i que ens ha servit, té diverses limitacions:

*   L’editor és lent i poc flexible comparat amb eines locals com VS Code o editors Markdown.
*   Gestionar arxius i carpetes és més feixuc i poc eficient.

Per això començarem a treballar com es fa realment al món professional: **treball local amb git** + **repositori remot a GitHub**. Git ja existia abans que GitHub i és un sistema de control de versions distribuït creat per Linus Torvalds.

## 💻 Com treballarem

1.  Partirem sempre d’un repositori creat a GitHub.
2.  Ens descarregarem una còpia local (clonar el repo).
3.  Tots els canvis es faran *en local* seguint el flux habitual:
    *   `git add`
    *   `git commit`
4.  Quan acabem la feina, pujarem els canvis a GitHub amb `git push`.
5.  Quan reprenguem la feina, farem `git pull` per tenir la versió actualitzada, especialment si treballem en diversos ordinadors (escola/casa).
